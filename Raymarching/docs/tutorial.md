## 1. 整体架构：一张全屏三角形上的光线追踪

传统 3D 渲染是把三角形网格投影到屏幕上。这个 shader 完全不同——它**没有任何 3D 模型**。宿主程序只做了一件事：画一个覆盖整个屏幕的三角形，然后把所有渲染工作交给 fragment shader。

每个像素执行的逻辑是：

```
对于屏幕上的每个像素:
    1. 根据像素坐标，计算从相机出发的一条射线
    2. 沿着这条射线在场景中"步进"，找到射线与场景几何体的交点
    3. 在交点处计算光照（法线、阴影、AO、多光源）
    4. 输出最终颜色
```

这就是所谓的 **Ray Marching**——一种运行在 GPU 上的光线追踪变体。它不需要三角形网格，而是用数学函数来描述场景中的所有几何体。

## 2. SDF 核心概念：用距离描述几何体

### 什么是 SDF

SDF（Signed Distance Function，有符号距离函数）是一个函数 `f(p)`，给定空间中任意一点 `p`，返回该点到几何体表面的**最近距离**：

- `f(p) > 0`：点在几何体**外部**
- `f(p) = 0`：点在几何体**表面上**
- `f(p) < 0`：点在几何体**内部**

这个"有符号"就体现在正负号上——它不仅告诉你有多远，还告诉你在里面还是外面。

### 为什么 SDF 有用

SDF 是 Ray Marching 能够工作的基础。当我们沿射线步进时，SDF 的值告诉我们**至少可以安全前进多远而不会穿过任何表面**。这就是所谓的 "sphere tracing" 思想——以当前点的 SDF 值为半径画一个球，球内保证不会碰到任何表面，所以可以安全跳过这段距离。

### 组合操作

多个 SDF 可以通过简单的操作组合在一起：

```glsl
// 并集（Union）：取两个距离的最小值，相当于"两个物体都存在"
float opUnion(float d1, float d2) {
    return min(d1, d2);
}

// 交集（Intersection）：取最大值
float opIntersection(float d1, float d2) {
    return max(d1, d2);
}

// 差集（Subtraction）：从 d1 中挖去 d2
float opSubtraction(float d1, float d2) {
    return max(d1, -d2);
}
```

在本 shader 中，`opU` 函数就是并集操作的变体，它同时携带了材质 ID：

```glsl
vec2 opU(vec2 d1, vec2 d2) {
    return (d1.x < d2.x) ? d1 : d2;
}
```

`vec2` 的 `.x` 分量是距离，`.y` 分量是材质 ID。取距离更小的那个，等价于"哪个表面更近就显示哪个"。

## 3. 光线步进（Ray Marching）：如何找到交点

### 核心算法

光线步进的算法非常直观。给定射线起点 `ro`（ray origin）和方向 `rd`（ray direction），我们沿着射线一步步前进：

```
t = 起始距离
循环:
    当前位置 p = ro + rd * t
    d = SDF(p)              // 查询当前位置到最近表面的距离
    如果 d 足够小:
        找到交点，返回 t
    t += d                  // 安全地前进 d 的距离
    如果 t 超出最大距离:
        没有命中，返回背景
```

每一步前进的距离**恰好等于 SDF 的值**。这是 sphere tracing 的核心思想：SDF 保证了在这个距离内不可能碰到表面，所以可以大胆前进。在表面附近，SDF 值会越来越小，步进也随之越来越精细，自然地收敛到表面。

### 代码中的实现

```glsl
vec2 raycast(in vec3 ro, in vec3 rd) {
    vec2 res = vec2(-1.0, -1.0);  // 默认：未命中

    float tmin = 1.0;
    float tmax = 20.0;

    // 先处理地板（水平面 y=0 的解析解）
    float tp1 = (0.0 - ro.y) / rd.y;
    if (tp1 > 0.0) {
        tmax = min(tmax, tp1);
        res = vec2(tp1, 1.0);     // 材质 ID = 1.0 代表地板
    }

    // 场景包围盒测试：射线是否可能命中物体区域
    vec2 tb = iBox(ro - vec3(0.0, 0.4, -0.5), rd, vec3(2.5, 0.41, 3.0));
    if (tb.x < tb.y && tb.y > 0.0 && tb.x < tmax) {
        tmin = max(tb.x, tmin);
        tmax = min(tb.y, tmax);

        float t = tmin;
        for (int i = 0; i < 70 && t < tmax; i++) {
            vec2 h = map(ro + rd * t);    // 查询场景 SDF
            if (abs(h.x) < (0.0001 * t))  // 足够接近表面
            {
                res = vec2(t, h.y);        // 记录距离和材质
                break;
            }
            t += h.x;                      // 前进 SDF 距离
        }
    }

    return res;
}
```

几个关键细节：

**地板的特殊处理**：地板是 `y = 0` 平面，射线与水平面的交点可以直接用解析公式 `t = -ro.y / rd.y` 算出来，不需要步进。这比用 SDF 步进快得多。地板交点还用来限制 `tmax`——如果射线在到达物体之前就先打到地板了，就不需要继续步进了。

**收敛判定 `abs(h.x) < 0.0001 * t`**：注意阈值是 `0.0001 * t` 而不是固定值。越远的物体，允许的误差越大。这是因为远处的像素覆盖的空间区域更大，亚像素级别的精度没有意义，同时避免了远处物体因精度不足而找不到交点的问题。

**最大 70 次迭代**：这是一个性能与质量的平衡。大多数射线在 20-40 步内就能收敛。

**包围盒加速**：`iBox` 函数计算射线与一个轴对齐包围盒（AABB）的交点。只有射线确实会穿过物体所在区域时，才进入步进循环。这是一个非常重要的优化。

### iBox：射线-AABB 相交测试

```glsl
vec2 iBox(in vec3 ro, in vec3 rd, in vec3 rad) {
    vec3 m = 1.0 / rd;
    vec3 n = m * ro;
    vec3 k = abs(m) * rad;
    vec3 t1 = -n - k;
    vec3 t2 = -n + k;
    return vec2(max(max(t1.x, t1.y), t1.z),
                min(min(t2.x, t2.y), t2.z));
}
```

这是经典的 slab method。一个 AABB 可以看作三对平行平面（x 方向一对、y 方向一对、z 方向一对）的交集。对每对平面，射线各有一个进入时刻和离开时刻。射线进入 AABB 的时刻是"所有进入时刻的最大值"，离开的时刻是"所有离开时刻的最小值"。如果进入 < 离开，说明射线穿过了这个盒子。

## 4. 三个代表性几何体的 SDF

我们选取三个从简到繁的几何体来讲解 SDF 的设计思路。

### 4.1 球体（Sphere）

```glsl
float sdSphere(vec3 p, float s) {
    return length(p) - s;
}
```

这是最简单的 SDF。一个以原点为中心、半径为 `s` 的球体。`length(p)` 是点 `p` 到原点的距离，减去半径 `s` 就得到了到球面的有符号距离：

- 在球外，`length(p) > s`，结果为正
- 在球面上，`length(p) == s`，结果为零
- 在球内，`length(p) < s`，结果为负

要把球放到任意位置 `center`，只需在调用时传入 `p - center`：

```glsl
sdSphere(pos - vec3(1.0, 0.5, 0.0), 0.25)  // 中心在 (1, 0.5, 0)，半径 0.25
```

这个"平移等价于坐标偏移"的技巧贯穿整个 shader，所有几何体都是围绕原点定义的，通过 `p - offset` 来放置到场景中的任意位置。

### 4.2 盒子（Box）

```glsl
float sdBox(vec3 p, vec3 b) {
    vec3 d = abs(p) - b;
    return min(max(d.x, max(d.y, d.z)), 0.0) + length(max(d, 0.0));
}
```

参数 `b` 是盒子的半边长（half-extents）。这个函数的思路：

**第一步 `abs(p)`**：利用盒子的对称性，把问题简化到第一象限（`x >= 0, y >= 0, z >= 0`）。无论 `p` 在哪个象限，取绝对值后等价于在第一象限的情况。

**第二步 `abs(p) - b`**：`d` 的每个分量表示"该轴方向上超出盒子表面多远"。如果 `d.x < 0`，说明在 x 方向上还在盒子内部。

**第三步的两部分**：
- `min(max(d.x, max(d.y, d.z)), 0.0)` —— 处理点在盒子**内部**的情况。当所有分量都为负时（在盒子内），取最大的那个负值（即到最近表面的距离，带负号）。`min(..., 0.0)` 确保在外部时这部分为零。
- `length(max(d, 0.0))` —— 处理点在盒子**外部**的情况。`max(d, 0.0)` 把负分量截为零（那些方向上还在盒子范围内），然后取向量长度，就得到了到盒子最近点的欧几里得距离。

两部分相加，始终有且仅有一部分非零，完美覆盖了内外两种情况。

### 4.3 圆环（Torus）

```glsl
float sdTorus(vec3 p, vec2 t) {
    return length(vec2(length(p.xz) - t.x, p.y)) - t.y;
}
```

参数 `t.x` 是圆环的主半径（中心到管道中心的距离），`t.y` 是管道半径。

这里的关键技巧是**降维**：

1. `length(p.xz)` —— 先忽略 y 轴，计算 `p` 在 xz 平面上到 y 轴的距离。圆环的中心线是 xz 平面上的一个圆，这一步把 3D 问题降成了 2D：不管你在圆环周围的哪个角度，只要到 y 轴的水平距离相同，情况就一样。
2. `length(p.xz) - t.x` —— 减去主半径，得到"到中心圆环的水平距离"。
3. `vec2(..., p.y)` —— 现在用这个水平距离和 y 坐标组成一个 2D 向量。这个向量描述了"在圆环截面上的位置"。
4. `length(...) - t.y` —— 到截面中心的距离减去管道半径，就是到圆环表面的有符号距离。

本质上就是：先把到圆环中心线的距离算出来（一个 2D 距离），然后对着这个中心线再算一个圆的 SDF。这种"嵌套降维"的思路在很多复杂 SDF 中都会出现。

### 小结

这三个例子展示了 SDF 设计的常见思路：
- **球体**：直接利用到中心的距离
- **盒子**：利用对称性（`abs`）把问题简化到一个象限，然后分内外讨论
- **圆环**：利用旋转对称性降维，把 3D 问题化成 2D

shader 中其他 20 多种几何体（胶囊、锥体、棱柱、金字塔等）都是这些基本思路的延伸和组合。如果你感兴趣，iq 的网站 [https://iquilezles.org/articles/distfunctions](https://iquilezles.org/articles/distfunctions) 有每一种的详细说明和可视化。

## 5. 场景构建：map 函数与包围盒加速

### map 函数：场景的完整描述

`map(pos)` 是整个渲染器的核心——它定义了"在空间中任意一点，离最近表面有多远，那个表面是什么材质"。返回值是 `vec2(距离, 材质ID)`。

```glsl
vec2 map(in vec3 pos) {
    vec2 res = vec2(pos.y, 0.0);  // 初始值：到 y=0 地板的距离

    // 第一组物体的包围盒测试
    if (sdBox(pos - vec3(-2.0, 0.3, 0.25), vec3(0.3, 0.3, 1.0)) < res.x) {
        res = opU(res, vec2(sdSphere(pos - vec3(-2.0, 0.25, 0.0), 0.25), 26.9));
        res = opU(res, vec2(sdRhombus(...), 17.0));
    }

    // 第二组、第三组、第四组、第五组... 类似结构
    // ...

    return res;
}
```

**结构模式**：物体按空间位置分成 5 组，每组用一个包围盒包裹。

**初始值 `vec2(pos.y, 0.0)`**：`pos.y` 就是到 `y = 0` 地板平面的距离（`sdPlane` 函数的简化形式）。材质 ID 为 `0.0`。

**opU 链式调用**：每个物体通过 `opU`（即 `min`）与当前结果合并，实现并集。最终 `res.x` 是到所有表面中最近的那个距离，`res.y` 是对应的材质 ID。

### 包围盒加速

注意每组物体外面都包了一个 `if (sdBox(...) < res.x)` 判断。这是性能优化的关键。

`sdBox(pos - groupCenter, groupExtents)` 计算的是当前点到这组物体的包围盒的距离。如果这个距离已经大于当前已知的最近距离 `res.x`，说明这组物体里不可能有比当前更近的表面了，**整组跳过**。

这使得每次 `map()` 调用时，不需要计算所有 25 个几何体的 SDF，可能只需要计算其中少数几个。在 70 次循环的步进过程中，这个优化的累计效果非常显著。

### 材质 ID 的设计

每个物体的材质 ID 是一个浮点数（如 `26.9`、`17.0`、`55.0`）。这些数字后续在着色时用来生成颜色：

```glsl
col = 0.2 + 0.2 * sin(m * 2.0 + vec3(0.0, 1.0, 2.0));
```

不同的材质 ID 经过 `sin` 函数后产生不同的 RGB 值，实现了每个物体有不同颜色的效果。这是一个巧妙的技巧——不需要为每个物体定义颜色，只需要一个数字。

## 6. 法线计算：从 SDF 中提取表面朝向

法线（Normal）是光照计算必需的。传统渲染中法线存储在顶点数据里，但 SDF 场景没有顶点。好消息是，SDF 的**梯度方向**就是法线方向。

直觉上：SDF 增长最快的方向就是"离开表面最快"的方向，也就是法线方向。

### 数值梯度法

代码用有限差分来近似梯度：

```glsl
vec3 calcNormal(in vec3 pos) {
    vec3 n = vec3(0.0);
    for (int i = ZERO; i < 4; i++) {
        vec3 e = 0.5773 * (2.0 * vec3((((i+3)>>1)&1), ((i>>1)&1), (i&1)) - 1.0);
        n += e * map(pos + 0.0005 * e).x;
    }
    return normalize(n);
}
```

这段代码看起来有点晦涩，实际上它做的事情等价于更直观的四面体差分方法。循环中 `i` 从 0 到 3，位运算生成的 4 个向量 `e` 分别指向正四面体的 4 个顶点方向：

```
e0 = 0.5773 * (-1, -1, -1) 的某个排列
e1 = 0.5773 * (-1,  1,  1) 的某个排列
e2 = 0.5773 * ( 1, -1,  1) 的某个排列
e3 = 0.5773 * ( 1,  1, -1) 的某个排列
```

`0.5773 ≈ 1/√3`，使得每个向量是单位向量。把 `pos` 沿这 4 个方向偏移微小距离 `0.0005`，查询 SDF 值，然后加权求和再归一化——这在数学上等价于计算梯度。

**为什么用循环而不是直接写 4 行？** 注释里说了：防止编译器把 `map()` 内联 4 次，导致编译后的 shader 代码膨胀。`map()` 函数体积很大（包含所有几何体），内联 4 次会严重影响性能。用循环可以让编译器只保留一份 `map()` 的代码。

**地板的特殊处理**：地板法线固定是 `(0, 1, 0)`（向上），不需要计算：

```glsl
vec3 nor = (m < 1.5) ? vec3(0.0, 1.0, 0.0) : calcNormal(pos);
```

材质 ID < 1.5 的就是地板（ID = 1.0），直接用常量法线，省去 4 次 `map()` 调用。

## 7. 软阴影（Soft Shadow）

硬阴影只有"全亮"和"全黑"两种状态，看起来非常生硬。现实中由于光源有面积，阴影边缘是柔和过渡的。SDF 提供了一种非常优雅的方式来近似软阴影。

### 原理

从表面点 `ro` 向光源方向 `rd` 发射一条阴影射线。沿着这条射线步进时，每一步的 SDF 值 `h` 不仅告诉我们离表面多远，还暗示了"光线有多少被遮挡"：

- 如果 `h` 很大 —— 附近没有遮挡物，完全照亮
- 如果 `h` 很小但不是零 —— 有物体擦边而过，产生半影（penumbra）
- 如果 `h` 接近零 —— 被遮挡，全黑

关键洞察：`h/t`（SDF 值除以已行进距离）的比值近似了"从光源看，遮挡物的张角"。比值越小说明有物体非常接近光线路径，阴影越浓。

### 代码实现

```glsl
float calcSoftshadow(in vec3 ro, in vec3 rd, in float mint, in float tmax) {
    // 限制步进范围不超过场景上方
    float tp = (0.8 - ro.y) / rd.y;
    if (tp > 0.0) tmax = min(tmax, tp);

    float res = 1.0;  // 1.0 = 完全照亮
    float t = mint;
    for (int i = ZERO; i < 24; i++) {
        float h = map(ro + rd * t).x;
        float s = clamp(8.0 * h / t, 0.0, 1.0);
        res = min(res, s);
        t += clamp(h, 0.01, 0.2);
        if (res < 0.004 || t > tmax) break;
    }
    res = clamp(res, 0.0, 1.0);
    return res * res * (3.0 - 2.0 * res);  // smoothstep 平滑
}
```

逐行分析：

**`res = 1.0`**：初始假设完全照亮。

**`s = clamp(8.0 * h / t, 0.0, 1.0)`**：这就是软阴影的核心公式。`h/t` 是我们说的"张角近似"，乘以 `8.0` 控制阴影的软硬程度——系数越大阴影越硬，越小阴影越软。

**`res = min(res, s)`**：取所有步进中最暗的值。只要路径上任何一点有遮挡，最终就变暗。

**`t += clamp(h, 0.01, 0.2)`**：步进距离被钳制在 `[0.01, 0.2]` 范围内。下限 `0.01` 防止在极靠近表面时步长太小导致死循环；上限 `0.2` 防止步子太大跳过薄的遮挡物。

**`res * res * (3.0 - 2.0 * res)`**：这是 `smoothstep` 函数的展开形式，把线性的阴影值映射成更平滑的 S 形曲线，让阴影过渡更自然。

**`mint` 参数（代码中传入 `0.02`）**：起始偏移量，避免表面自遮挡（self-shadowing）。如果从表面精确位置开始步进，由于数值精度问题，SDF 可能返回接近零的值，导致自身产生阴影。偏移一小段距离即可解决。

## 8. 环境光遮蔽（Ambient Occlusion）

环境光遮蔽（AO）模拟的是"角落和缝隙处更暗"的效果。两面墙交汇的角落、物体底部接近地面的区域，由于被周围的表面遮挡了更多的环境光，看起来更暗。

### 原理

SDF 让 AO 的计算变得非常简单。思路是：从表面沿法线方向取若干采样点，比较"采样点到表面的实际距离"和"沿法线的理论距离"：

- 如果两者相等 —— 法线方向上没有遮挡（开阔区域）
- 如果实际距离 < 理论距离 —— 说明有其他物体挡在法线方向附近（角落或凹陷）

### 代码实现

```glsl
float calcAO(in vec3 pos, in vec3 nor) {
    float occ = 0.0;
    float sca = 1.0;
    for (int i = ZERO; i < 5; i++) {
        float h = 0.01 + 0.12 * float(i) / 4.0;  // 采样距离：0.01, 0.04, 0.07, 0.10, 0.13
        float d = map(pos + h * nor).x;            // 该点到最近表面的实际距离
        occ += (h - d) * sca;                       // 累加差值（理论距离 - 实际距离）
        sca *= 0.95;                                 // 逐渐降低远处采样的权重
        if (occ > 0.35) break;                       // 已经足够暗了，提前退出
    }
    return clamp(1.0 - 3.0 * occ, 0.0, 1.0) * (0.5 + 0.5 * nor.y);
}
```

**5 个采样点**：沿法线方向均匀分布在 `0.01` 到 `0.13` 的距离上。

**`h - d`**：如果没有遮挡，`d` 应该等于 `h`（采样点在空旷处，到最近表面的距离就等于离出发点的距离）。差值越大说明遮挡越严重。

**`sca *= 0.95`**：越远的采样权重越低。近处的遮挡对 AO 的影响更大。

**`(0.5 + 0.5 * nor.y)`**：这是一个额外的调整——朝上的表面（`nor.y` 接近 1）更亮，朝下的更暗。模拟了"天空来自上方"的环境光分布。

整个计算只需要 5 次 `map()` 调用，非常高效，但效果已经很好了。

## 9. 光照模型：四个光源的组合

这个 shader 使用了 4 个"虚拟光源"来模拟丰富的照明效果。每个光源不需要是真实的点光源或方向光，而是通过不同的着色策略来模拟不同方向来的光。

### 9.1 太阳光（Sun / Key Light）

```glsl
{
    vec3  lig = normalize(vec3(-0.5, 0.4, -0.6));      // 光照方向
    vec3  hal = normalize(lig - rd);                     // 半程向量
    float dif = clamp(dot(nor, lig), 0.0, 1.0);         // 漫反射（Lambert）
          dif *= calcSoftshadow(pos, lig, 0.02, 2.5);   // 乘以阴影
    float spe = pow(clamp(dot(nor, hal), 0.0, 1.0), 16.0); // Blinn-Phong 高光
          spe *= dif;                                    // 在阴影中无高光
          spe *= 0.04 + 0.96 * pow(clamp(1.0 - dot(hal, lig), 0.0, 1.0), 5.0);
    lin += col * 2.20 * dif * vec3(1.30, 1.00, 0.70);   // 暖色漫反射
    lin +=       5.00 * spe * vec3(1.30, 1.00, 0.70);   // 暖色高光
}
```

**漫反射（Diffuse）**：经典 Lambert 模型，`dot(nor, lig)` 是法线与光照方向的余弦值。面向光源的面亮，背向的面暗。

**阴影调制**：漫反射强度乘以 `calcSoftshadow` 的结果。被遮挡的区域，漫反射会被压暗。

**高光（Specular）**：使用 Blinn-Phong 模型。`hal` 是光照方向和视线方向的半程向量（half vector），`dot(nor, hal)` 的幂次越高，高光越锐利。这里指数是 16。

**Fresnel 项**：`0.04 + 0.96 * pow(1.0 - dot(hal, lig), 5.0)` 是 Schlick 近似的 Fresnel 反射率。它让高光在掠射角（grazing angle）时更强，这是真实物理中的反射行为。`0.04` 是大多数非金属材料的基础反射率。

**光照颜色 `vec3(1.30, 1.00, 0.70)`**：偏暖的橙黄色，模拟阳光。

**注意漫反射和高光的处理差异**：漫反射是 `col * 2.20 * dif * lightColor`，高光是 `5.00 * spe * lightColor`。漫反射乘以了物体本身颜色 `col`（因为漫反射光被材质吸收后着色），高光没乘 `col`（因为高光反射的是光源颜色本身）。

### 9.2 天空光（Sky Light）

```glsl
{
    float dif = sqrt(clamp(0.5 + 0.5 * nor.y, 0.0, 1.0));  // 半球漫反射
          dif *= occ;                                          // 乘以 AO
    float spe = smoothstep(-0.2, 0.2, ref.y);                 // 天空方向的反射
          spe *= dif;
          spe *= 0.04 + 0.96 * pow(clamp(1.0 + dot(nor, rd), 0.0, 1.0), 5.0);
          spe *= calcSoftshadow(pos, ref, 0.02, 2.5);
    lin += col * 0.60 * dif * vec3(0.40, 0.60, 1.15);   // 冷色调天空光
    lin +=       2.00 * spe * vec3(0.40, 0.60, 1.30);   // 天空反射高光
}
```

**`0.5 + 0.5 * nor.y`**：这是一个快速的半球光照近似。`nor.y = 1`（朝上）时值为 1（最亮），`nor.y = -1`（朝下）时值为 0（最暗）。`sqrt` 让过渡更柔和。

**乘以 AO**：天空光是环境光的一种，所以被 AO 调制。在角落和凹陷处，天空光更弱。

**天空反射高光**：`ref` 是视线关于法线的反射方向，`ref.y > 0` 说明反射向上（朝天空），用 `smoothstep` 做了平滑过渡。这模拟了光滑表面反射天空的效果。

**天空光颜色 `vec3(0.40, 0.60, 1.15)`**：冷色调蓝色，与太阳的暖色形成对比，让画面色彩更丰富。

### 9.3 背光（Back Light / Bounce Light）

```glsl
{
    float dif = clamp(dot(nor, normalize(vec3(0.5, 0.0, 0.6))), 0.0, 1.0)
              * clamp(1.0 - pos.y, 0.0, 1.0);
          dif *= occ;
    lin += col * 0.55 * dif * vec3(0.25, 0.25, 0.25);
}
```

**方向 `(0.5, 0.0, 0.6)`**：一个水平方向的补光，模拟地面对光线的反弹（bounce light）。

**`clamp(1.0 - pos.y, 0.0, 1.0)`**：越靠近地面（`pos.y` 接近 0）反弹光越强。远离地面的物体几乎不受影响。这在物理上是合理的——地面反弹的光主要影响靠近地面的物体下方。

**灰色 `vec3(0.25, 0.25, 0.25)`**：背光颜色是中性灰，强度较低（`0.55`），作为填充光避免暗部全黑。

### 9.4 次表面散射近似（SSS / Subsurface Scattering）

```glsl
{
    float dif = pow(clamp(1.0 + dot(nor, rd), 0.0, 1.0), 2.0);
          dif *= occ;
    lin += col * 0.25 * dif * vec3(1.00, 1.00, 1.00);
}
```

**`1.0 + dot(nor, rd)`**：`dot(nor, rd)` 在法线和视线方向垂直时为 0，在边缘处（视线几乎平行于表面）为 -1 附近。所以 `1.0 + dot(nor, rd)` 在边缘为 0，在掠射角增大时变大……不对，让我重新理解。

`rd` 是从相机指向场景的方向，`nor` 是表面法线。`dot(nor, rd)` 在法线背向相机时为负值（正常的面朝相机的表面），值越接近 -1 表示越正面朝向相机。`1.0 + dot(nor, rd)` 在正对相机时接近 0，在**轮廓边缘**（法线几乎垂直于视线）时接近 1。

所以这个效果在物体边缘处最亮——这就是 Fresnel 效应 / 边缘光（rim light）的简单近似。它模拟了光线穿透物体边缘薄处的次表面散射效果，给物体一种微妙的通透感。

### 四个光源的协作

| 光源 | 方向 | 色调 | 受 AO 影响 | 受阴影影响 | 作用 |
|------|------|------|-----------|-----------|------|
| 太阳 | 固定方向 | 暖橙 | 否 | 是 | 主光源，定义明暗关系 |
| 天空 | 上方半球 | 冷蓝 | 是 | 是（反射） | 填充暗部，增加色彩对比 |
| 背光 | 水平方向 | 灰色 | 是 | 否 | 填充底部，防止全黑 |
| SSS  | 边缘 | 白色 | 是 | 否 | 边缘高亮，增加通透感 |

这种组合方式在电影和游戏行业中很常见，被称为"三点布光"（key / fill / rim）的变体。太阳是 key light，天空是 fill light，SSS 是 rim light，背光是额外的 bounce light。

## 10. 材质与棋盘格纹理

### 物体材质

材质着色非常简洁：

```glsl
col = 0.2 + 0.2 * sin(m * 2.0 + vec3(0.0, 1.0, 2.0));
```

`m` 是材质 ID（浮点数），`sin` 的三个分量分别偏移 `0`、`1`、`2` 弧度，让 R、G、B 有不同相位。不同的 `m` 值产生不同颜色。`0.2 + 0.2 * sin(...)` 把颜色限制在 `[0.0, 0.4]` 范围内——偏暗淡的色调，让光照效果更突出。

### 地板棋盘格

地板使用了带抗锯齿的棋盘格纹理：

```glsl
if (m < 1.5) {
    vec3 dpdx = ro.y * (rd / rd.y - rdx / rdx.y);
    vec3 dpdy = ro.y * (rd / rd.y - rdy / rdy.y);
    float f = checkersGradBox(3.0 * pos.xz, 3.0 * dpdx.xz, 3.0 * dpdy.xz);
    col = 0.15 + f * vec3(0.05);
    ks = 0.4;
}
```

**像素足迹投影**：`dpdx` 和 `dpdy` 是"屏幕上相邻像素对应的地板位置差异"（像素在地板上的覆盖范围）。这需要 `rdx` 和 `rdy`——相邻像素的射线方向，也就是"射线微分"（ray differentials）。

**为什么需要射线微分？** 如果直接用 `mod` 或 `fract` 做棋盘格，远处的棋盘格会变得非常小，一个像素覆盖多个黑白格子，产生严重的摩尔纹（aliasing）。通过知道像素的覆盖范围，可以在一个像素覆盖多个格子时做平均——这就是抗锯齿滤波。

**checkersGradBox 函数**：

```glsl
float checkersGradBox(in vec2 p, in vec2 dpdx, in vec2 dpdy) {
    vec2 w = abs(dpdx) + abs(dpdy) + 0.001;   // 滤波核宽度
    vec2 i = 2.0 * (abs(fract((p - 0.5*w) * 0.5) - 0.5)
                   - abs(fract((p + 0.5*w) * 0.5) - 0.5)) / w;
    return 0.5 - 0.5 * i.x * i.y;
}
```

这是棋盘格的**解析盒式滤波**（analytical box filter）。它不是逐像素采样，而是精确计算"这个像素覆盖区域内棋盘格的平均值"。远处像素覆盖范围大时，黑白格子被平均成灰色，自然消除了摩尔纹。

## 11. 相机系统

### 相机矩阵

```glsl
mat3 setCamera(in vec3 ro, in vec3 ta, float cr) {
    vec3 cw = normalize(ta - ro);                    // 前方（look direction）
    vec3 cp = vec3(sin(cr), cos(cr), 0.0);           // 世界上方向（可旋转）
    vec3 cu = normalize(cross(cw, cp));               // 相机右方
    vec3 cv = cross(cu, cw);                          // 相机上方
    return mat3(cu, cv, cw);
}
```

这是标准的 "lookAt" 相机构建。给定相机位置 `ro`、目标点 `ta`、和滚转角 `cr`，构建一个 3x3 旋转矩阵，把相机空间的方向转换到世界空间。`cr` 在本 shader 中为 `0.0`，即相机没有滚转。

### 相机动画

```glsl
vec3 ta = vec3(0.25, -0.75, -0.75);         // 目标点（固定）
vec3 ro = ta + vec3(4.5 * cos(0.1*time + 7.0*mo.x),
                    2.2,
                    4.5 * sin(0.1*time + 7.0*mo.x));
```

相机围绕目标点做圆周运动：
- `4.5` 是轨道半径
- `2.2` 是固定高度
- `0.1 * time` 让相机随时间自动旋转
- `7.0 * mo.x` 让鼠标可以控制旋转角度

### 从像素到射线

```glsl
vec2 p = (2.0 * fragCoord - iResolution.xy) / iResolution.y;
const float fl = 2.5;
vec3 rd = ca * normalize(vec3(p, fl));
```

**像素坐标归一化**：`(2.0 * fragCoord - iResolution.xy) / iResolution.y` 把像素坐标从 `[0, width] x [0, height]` 映射到以画面中心为原点、y 范围为 `[-1, 1]` 的坐标系。除以 `iResolution.y`（而不是 `.x`）保证了不管窗口长宽比如何，y 方向总是 `[-1, 1]`，不会变形。

**焦距 `fl = 2.5`**：`normalize(vec3(p, fl))` 构建了从原点指向 `(p.x, p.y, fl)` 的方向向量。`fl` 越大，视野越窄（像望远镜）；越小，视野越宽（像广角）。`fl = 2.5` 大约对应 ~44° 的垂直视场角（`2 * atan(1/2.5) ≈ 43.6°`）。

**`ca * ...`**：用相机矩阵把局部射线方向转到世界空间。

## 12. 雾效（大气透视）

```glsl
col = mix(col, vec3(0.7, 0.7, 0.9), 1.0 - exp(-0.0001 * t * t * t));
```

这一行实现了大气透视 / 距离雾效。`t` 是射线行进的距离：

- **`exp(-0.0001 * t³)`**：指数衰减函数，距离越远值越接近 0
- **`1.0 - exp(...)`**：反转后，距离越远值越接近 1
- **`mix(col, fogColor, factor)`**：把物体颜色和雾色（`vec3(0.7, 0.7, 0.9)`，浅蓝灰色）按距离混合

近处物体几乎不受影响，远处物体逐渐融入背景色。使用 `t³`（立方）而不是 `t`（线性），让雾效在近处非常微弱，只在中远距离开始明显——这更符合真实大气散射的非线性特性。

背景色也设置成了相近的色调：

```glsl
vec3 col = vec3(0.7, 0.7, 0.9) - max(rd.y, 0.0) * 0.3;
```

背景从下方的 `(0.7, 0.7, 0.9)` 到上方的 `(0.4, 0.4, 0.6)` 渐变，模拟天空。雾色和天空底部颜色一致，确保远处物体能自然融入背景。

## 13. Gamma 校正与超采样抗锯齿

### Gamma 校正

```glsl
col = pow(col, vec3(0.4545));
```

`0.4545 ≈ 1/2.2`，这是标准的 sRGB gamma 校正。

显示器的物理特性是非线性的——输入值 0.5 显示出来的亮度不是 50%，而是大约 21.8%（`0.5^2.2 ≈ 0.218`）。Gamma 校正通过预先做一次反向变换来补偿这个非线性。

整个 shader 的所有光照计算都在**线性空间**中进行（这很重要——光照运算在线性空间中才物理正确），最后输出前做一次 gamma 校正。

### 超采样抗锯齿（SSAA）

```glsl
#define AA 2

for (int m = ZERO; m < AA; m++)
for (int n = ZERO; n < AA; n++) {
    vec2 o = vec2(float(m), float(n)) / float(AA) - 0.5;
    vec2 p = (2.0 * (fragCoord + o) - iResolution.xy) / iResolution.y;
    // ... 完整的渲染流程 ...
    tot += col;
}
tot /= float(AA * AA);
```

`AA = 2` 意味着每个像素被分成 2x2 = 4 个子样本。每个子样本偏移 `o`（分别是 `(-0.5,-0.5), (0,-0.5), (-0.5,0), (0,0)` 的归一化坐标），独立执行完整的渲染流程，最后取平均值。

这是最暴力但也最有效的抗锯齿方法——几何边缘、阴影边缘、高光边缘全都能受益。代价是渲染时间变为原来的 4 倍。`AA = 1` 可以关闭抗锯齿以获得更好的性能（通过 `HW_PERFORMANCE` 宏控制）。

**`ZERO` 宏**：`#define ZERO (min(iFrame, 0))`——永远等于 0，但编译器不知道。这防止编译器在编译期知道循环次数，避免过度展开循环导致的代码膨胀和编译时间过长。

## 14. 完整渲染流程回顾

让我们把所有模块串联起来，回顾每个像素的完整渲染过程：

```
mainImage(fragCoord)
│
├── 1. 计算相机位置和朝向（随时间旋转，鼠标可控）
│
├── 2. 对每个子采样（AA x AA 次）:
│   │
│   ├── 2a. 根据像素坐标 + 子采样偏移，计算射线方向 rd
│   │       同时计算相邻像素的射线方向 rdx, rdy（用于纹理抗锯齿）
│   │
│   ├── 2b. raycast(ro, rd) —— 射线与场景求交
│   │   ├── 先解析求解地板交点
│   │   ├── 场景总包围盒测试（iBox），如果射线不穿过物体区域直接返回
│   │   └── 循环最多 70 次 sphere tracing
│   │       └── 每步调用 map(p) 获取最近距离和材质
│   │           └── map 内部：5 组包围盒加速 + 25 种 SDF
│   │
│   ├── 2c. 如果命中：
│   │   ├── 计算法线 calcNormal（4 次 map 调用）
│   │   ├── 计算反射方向 ref
│   │   ├── 根据材质 ID 确定基础颜色（或棋盘格纹理）
│   │   ├── 计算 AO（5 次 map 调用）
│   │   └── 4 个光源叠加：
│   │       ├── 太阳光：Lambert 漫反射 + Blinn-Phong 高光 + 软阴影（24 次 map）
│   │       ├── 天空光：半球漫反射 + 反射高光 * AO + 反射方向阴影（24 次 map）
│   │       ├── 背光：填充光 * AO
│   │       └── SSS：边缘光 * AO
│   │
│   ├── 2d. 应用雾效（距离混合）
│   │
│   └── 2e. Gamma 校正
│
└── 3. 子采样结果取平均，输出最终颜色
```

单个像素最多可能调用 `map()` 的次数：70（步进）+ 4（法线）+ 5（AO）+ 24（太阳阴影）+ 24（天空反射阴影）= **127 次**。再乘以 4 个子采样 = **508 次 `map()` 调用/像素**。这就是为什么 `map()` 内部的包围盒优化如此重要。

## 15. 动手实践：如何从零开始

如果你想从零开始做一个类似的 SDF 渲染效果，建议按以下顺序逐步构建：

### 第一步：最小可运行的 Ray Marching

先实现一个只有地板和球体的场景，不带任何花哨效果：

```glsl
float map(vec3 p) {
    float ground = p.y;
    float sphere = length(p - vec3(0.0, 0.5, 0.0)) - 0.5;
    return min(ground, sphere);
}

void mainImage(out vec4 fragColor, in vec2 fragCoord) {
    // 固定相机
    vec3 ro = vec3(0.0, 1.0, 3.0);
    vec3 ta = vec3(0.0, 0.5, 0.0);

    // 简单的 lookAt
    vec3 cw = normalize(ta - ro);
    vec3 cu = normalize(cross(cw, vec3(0, 1, 0)));
    vec3 cv = cross(cu, cw);

    vec2 p = (2.0 * fragCoord - iResolution.xy) / iResolution.y;
    vec3 rd = normalize(p.x * cu + p.y * cv + 2.0 * cw);

    // ray marching
    float t = 0.0;
    for (int i = 0; i < 64; i++) {
        float d = map(ro + rd * t);
        if (d < 0.001) break;
        t += d;
        if (t > 20.0) break;
    }

    // 简单的法线着色
    vec3 col = vec3(0.7, 0.7, 0.9);
    if (t < 20.0) {
        vec3 pos = ro + rd * t;
        vec2 e = vec2(0.001, 0.0);
        vec3 nor = normalize(vec3(
            map(pos + e.xyy) - map(pos - e.xyy),
            map(pos + e.yxy) - map(pos - e.yxy),
            map(pos + e.yyx) - map(pos - e.yyx)
        ));
        col = 0.5 + 0.5 * nor;  // 法线可视化
    }

    fragColor = vec4(col, 1.0);
}
```

这个版本大约 30 行代码，但已经能渲染出一个球体在地板上的 3D 效果。

### 第二步：添加基本光照

把法线可视化替换成 Lambert 漫反射：

```glsl
vec3 lig = normalize(vec3(-0.5, 0.4, -0.6));
float dif = clamp(dot(nor, lig), 0.0, 1.0);
col = vec3(0.2) + vec3(0.8) * dif;
```

### 第三步：添加阴影

### 第四步：添加 AO

加入 `calcAO`，让角落变暗。

### 第五步：添加更多几何体
