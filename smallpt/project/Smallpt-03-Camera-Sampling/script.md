  Vec cy = (cx % cam.d).norm() * .5135; // 垂直方向

--- narration ---
smallpt 用的是最简单的 **针孔相机**
一个点（相机位置）和一个方向（朝向场景）
相机在 (50, 52, 295.6)，这个位置正对着 Cornell Box

这两个向量定义了成像平面的宽和高
每个像素对应成像平面上的一个点
从相机穿过这个点就形成一条 **主射线**

  一条线段横跨视锥宽度标注 "cx 长度 = 0.5135"。
  从相机到线段中点标注 "d 长度 = 1（归一化后）"。
  公式 "tan(FOV/2) = 0.5135 / 1 → FOV ≈ 54.2°"
下方注释，字号 24px，颜色 #8b949e：
  "w*.5135/h 保证了宽高比正确：1024/768 × 0.5135 ≈ 0.685"
  "cx 只有 x 分量，因为相机恰好面朝 -z 方向"

--- narration ---
0.5135 这个数字是怎么来的？
它是视场角的正切值的一半
**tan(FOV/2) = 0.5135**
反算得到视场角大约 **54.2 度**
cx 的长度乘以 w/h
保证了水平方向按宽高比缩放
这样 1024 乘 768 的图像不会变形
相机恰好面朝负 z 方向
所以 cx 只有 x 分量
cy 通过叉积自动获得正交的 y 方向

@exit: fade
@visual: image

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "2×2 子像素分层采样"，字号 48px，粗体，白色，距顶 50px。
中部为示意图，居中，宽度 50%：
  一个大正方形代表一个像素，内部被十字线分成 4 个小格。
  四个小格分别标注 "sx=0,sy=0"、"sx=1,sy=0"、"sx=0,sy=1"、"sx=1,sy=1"。
  每个小格内有若干随机散布的小点（采样点），标注 "samps 个采样"。
  大正方形外标注 "一个像素"。
下方代码块，背景 #161b22，字号 22px，宽度 85%：
  for (int sy = 0; sy < 2; sy++)     // 2 行子像素
    for (int sx = 0; sx < 2; sx++) { // 2 列子像素
      Vec r = Vec();
      for (int s = 0; s < samps; s++) {  // 每个子像素 samps 条路径
        // ... 生成射线并追踪 ...
        r = r + radiance(...) * (1./samps);
      }
      c[i] = c[i] + Vec(clamp(r.x),clamp(r.y),clamp(r.z)) * .25;
    }
底部注释 "总采样数 = samps × 4（每个子像素 samps 次）"，字号 22px，#8b949e。

--- narration ---
每个像素被分成 **2×2 的子像素网格**
每个子像素独立做 samps 次路径追踪
然后四个子像素取平均（乘以 0.25）
命令行参数除以 4 就是 samps
比如传入 1000，samps 等于 250
每个子像素采 250 条路径，总共 1000 条
为什么要分子像素？
因为每个子像素会用 **帐篷滤波器** 产生偏移
分区域采样比完全随机分布的点 **方差更低**
这叫 **分层采样 (Stratified Sampling)**
是蒙特卡洛加速收敛的经典技巧


>>> 帐篷滤波器 #B05
@enter: fade-up
@exit: fade
@visual: image

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "帐篷滤波器 (Tent Filter)"，字号 48px，粗体，白色，距顶 50px。
中部分为上下两部分：
上部为函数图像，居中，宽度 55%：
  x 轴标注 "-1 到 1"，y 轴标注 "概率密度"。
  画一个三角形（帐篷形状），顶点在 (0, 1)，底边从 -1 到 1。
  标注 "中心权重高，边缘权重低"。
  对比：旁边画一个矩形（均匀分布），标注 "Box filter（均匀）"。
下部代码块，背景 #161b22，字号 22px，宽度 80%：
  double r1 = 2*erand48(Xi);  // [0, 2) 均匀随机
  double dx = r1<1 ? sqrt(r1)-1 : 1-sqrt(2-r1);  // 帐篷采样
  double r2 = 2*erand48(Xi);
  double dy = r2<1 ? sqrt(r2)-1 : 1-sqrt(2-r2);
  // dx, dy ∈ [-1, 1]，三角形分布
下方注释，字号 22px，颜色 #8b949e：
  "逆变换采样：均匀随机数 → 三角形分布的偏移量"
  "效果：像素中心的采样点更密集，边缘稀疏"

--- narration ---
每个采样点在子像素内的位置不是均匀随机的
而是按 **帐篷分布（三角形分布）** 生成
帐篷分布在中心处概率密度最高
越靠近边缘越低
效果是像素 **中心附近的采样更密集**
这比均匀分布（box filter）产生的图像更锐利
因为远离像素中心的采样贡献被自然降权了
代码用了 **逆变换采样**
把 [0,2) 的均匀随机数映射到 [-1,1] 的三角形分布
r1 小于 1 时用 sqrt(r1)-1
r1 大于等于 1 时用 1-sqrt(2-r1)
两段拼接形成完整的帐篷形状


>>> 射线生成 #B06
@enter: fade-up
@exit: fade
@visual: image

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "从像素坐标到射线方向"，字号 48px，粗体，白色，距顶 50px。
代码块，背景 #161b22，圆角 12px，宽度 85%，字号 22px：
  Vec d = cx * ( ( (sx+.5 + dx)/2 + x) / w - .5) +
          cy * ( ( (sy+.5 + dy)/2 + y) / h - .5) + cam.d;
  r = r + radiance(Ray(cam.o + d*140, d.norm()), 0, Xi) * (1./samps);
下方逐步分解，字号 24px，行距 36px：
  "(sx+.5+dx)/2" — 子像素内的偏移量，映射到 [0,1] 子像素宽度内
  "(... + x)/w - .5" — 加上像素坐标，归一化到 [-0.5, 0.5]
  "cx * (...) + cy * (...)" — 用相机基向量缩放
  "+ cam.d" — 加上相机朝向
  "cam.o + d*140" — 射线起点偏移（近裁剪）
  "d.norm()" — 归一化得到射线方向
每行前有小箭头标记，关键变量 accent 色。

--- narration ---
这行代码把像素坐标转换成射线方向
先算子像素内的精确位置
sx 加 0.5 是子像素中心，dx 是帐篷偏移
除以 2 是因为一个像素有 2 个子像素
加上 x 后除以 w 再减 0.5
就把 [0, w] 映射到了 [-0.5, 0.5]
乘以 cx 和 cy 得到成像平面上的位移
加上相机朝向 cam.d 就是射线方向
注意射线起点不是 cam.o 本身
而是 cam.o 加上 d 乘以 140
这相当于把起点往前推了 140 个单位
效果是一个 **近裁剪面**
排除相机太近的物体


>>> 主渲染循环 #B07
@enter: fade-up
@exit: fade
@visual: image

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "主渲染循环"，字号 52px，粗体，白色，距顶 50px。
代码块，背景 #161b22，宽度 85%，字号 22px：
  int w=1024, h=768;
  int samps = argc==2 ? atoi(argv[1])/4 : 1;
  Vec *c = new Vec[w*h];  // 帧缓冲

  #pragma omp parallel for schedule(dynamic,1) private(r)
  for (int y = 0; y < h; y++) {
    fprintf(stderr, "\rRendering (%d spp) %5.2f%%", samps*4, 100.*y/(h-1));
    for (unsigned short x=0, Xi[3]={0,0,y*y*y}; x<w; x++)
      for (int sy=0, i=(h-y-1)*w+x; sy<2; sy++)
        for (int sx=0; sx<2; sx++, r=Vec()) {
          for (int s=0; s<samps; s++) { /* 采样 + 追踪 */ }
          c[i] = c[i] + Vec(clamp(r.x),clamp(r.y),clamp(r.z))*.25;
        }
  }
代码下方三个注解卡片横排，字号 22px：
  "OpenMP: 每行一个任务，dynamic 调度" — accent 色边框
  "Xi: 伪随机种子，y³ 让相邻行不相关" — accent 色边框
  "(h-y-1)*w+x: PPM 是从上到下，y 循环从下到上" — accent 色边框

--- narration ---
主循环按 **行** 遍历所有像素
**OpenMP** 的 pragma 让多核 CPU 并行处理不同行
schedule(dynamic,1) 表示每次分配一行给空闲线程
动态调度比静态好
因为不同行的渲染时间可能差异很大
Xi 是 erand48 的随机数种子
用 y 的立方初始化，让相邻行的随机序列不相关
像素索引用 **(h-y-1)*w+x**
因为 PPM 格式的第一行是图像顶部
而 y 循环是从 0（底部）开始
所以要翻转 y 坐标


>>> Gamma 校正与 PPM 输出 #B08

中部分为上下两部分：
上部代码块，背景 #161b22，字号 26px，宽度 75%：
  inline int toInt(double x) {
    return int(pow(clamp(x), 1/2.2) * 255 + .5);
  }
  // 输出 PPM
  FILE *f = fopen("image.ppm", "w");
  fprintf(f, "P3\n%d %d\n%d\n", w, h, 255);
  for (int i=0; i<w*h; i++)
    fprintf(f,"%d %d %d ", toInt(c[i].x), toInt(c[i].y), toInt(c[i].z));
下部分为左右对比：
左侧 "线性空间 → 偏暗"，下方一个偏暗的色带示意。
右侧 "Gamma 校正后 → 正确"，下方一个明度正确的色带示意。
箭头从左到右标注 "pow(x, 1/2.2)"。
底部注释 "PPM 是最简单的图像格式：纯文本，每像素三个整数 (R G B)"，字号 22px，#8b949e。

--- narration ---
所有光照计算都在 **线性空间** 中进行
线性空间的值直接显示在屏幕上会偏暗
因为显示器有 gamma 2.2 的非线性响应
toInt 函数做了 **pow(x, 1/2.2)** 的校正
然后乘以 255 转成 0-255 的整数
clamp 确保颜色值在 0 到 1 之间
加 0.5 是为了 **四舍五入**
输出格式是 PPM
纯文本格式，极其简单
文件头写分辨率和最大值
然后逐像素写三个整数
可以用 xv 或任何图像查看器打开

右栏注释 "缺点: 只对球有效，其他形状需不同公式"，字号 22px，#8b949e

--- narration ---
Raymarching 用 **Sphere Tracing** 步进求交
每步查询 SDF 值，安全前进
最多 70 次迭代才收敛到表面
优点是对任何 SDF 都通用
缺点是有迭代开销，且可能漏掉极薄的物体
smallpt 用 **解析求交**
对球体直接解二次方程
一步就得到精确的交点距离 t
没有迭代，没有近似误差
但这个公式只对球有效
如果场景有三角形网格，需要完全不同的求交代码
