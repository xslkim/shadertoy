>>> 渲染方程简介 #B04
@enter: fade-up
@exit: fade
@visual: image

--- visual ---

  L_o(x, ω_o) = L_e(x, ω_o) + ∫_Ω f_r(x, ω_i, ω_o) · L_i(x, ω_i) · cos θ_i dω_i
  字号 36px，等宽字体，白色，关键符号用 accent 色高亮。
方程下方 40px，四行解释，左对齐，字号 28px，颜色 #e6edf3：
  "L_o = 出射辐射度（我们要算的）" — L_o 用 accent 色
  "L_e = 自发光（只有光源才有）" — L_e 用 #ff7b72
  "f_r = BRDF（材质如何反射光）" — f_r 用 #a5d6ff
  "∫ L_i · cosθ = 来自所有方向的入射光的加权积分" — ∫ 用 accent 色
底部注释 "这个积分没有解析解，必须用数值方法（蒙特卡洛）近似"，字号 22px，颜色 #8b949e。

--- narration ---
路径追踪的数学基础是 **渲染方程**
1986 年由 Kajiya 提出
它说：一个点向某方向发出的光
等于自发光，加上来自 **所有方向** 的入射光的积分
积分里面有三个东西相乘
入射光 L_i，BRDF f_r，还有一个余弦权重
这个积分覆盖整个半球，维度很高
而且 L_i 本身又是另一个点的出射光
形成 **递归** 结构
解析求解是不可能的
所以我们用 **蒙特卡洛方法** 来近似

右栏：数学公式风格：
  第一行 "精确积分: ∫ f(x) dx — 不可能算"，字号 28px
  第二行 "MC 估计: (1/N) · Σ f(x_i) / p(x_i)"，字号 28px，accent 色
  第三行 "N 越大，估计越准确"，字号 24px，颜色 #8b949e
  第四行 "误差 ∝ 1/√N — 4倍采样 = 2倍质量"，字号 24px，颜色 #8b949e
底部 "关键洞察：不需要遍历所有方向，只需要随机采样足够多次"，字号 26px，白色。

--- narration ---
蒙特卡洛积分的核心思想非常简单
要算一个函数的积分
我们不需要遍历定义域上的每一个点
只需要 **随机采样** 若干个点
用这些点的函数值的平均来 **估计** 积分
采样越多，估计越接近真实值
但收敛速度是 **1 除以根号 N**
也就是说，要让噪声减半，需要 4 倍的采样量
这就是为什么路径追踪的图，采样少时非常嘈杂
采样多了才慢慢变干净

@enter: fade-up

--- visual ---

顶部标题 "射线-球体求交"，字号 52px，粗体，白色，距顶 50px。
中部左侧为几何示意图（占 45%）：
  一个圆（球的截面），圆心标注 "p"，半径线标注 "r"。
  一条射线从左下角穿过圆，起点标注 "o"，方向箭头标注 "d"。
  射线与圆有两个交点，标注 "t1" 和 "t2"。
  从圆心向射线做垂线，垂足标注 "H"，垂线长度标注 "h"。
  o 到 H 的距离标注 "b = op · d"。
中部右侧为公式推导（占 45%），字号 26px：
  第一行 "球面方程: |P - p|² = r²"
  第二行 "代入 P = o + t·d:"
  第三行 "|o + t·d - p|² = r²"
  第四行 "令 op = p - o:"
  第五行 "t² - 2(op·d)t + (op·op - r²) = 0"
  第六行 "b = op·d,  det = b² - op·op + r²"
  第七行 accent 色: "t = b ± √det"

--- narration ---
球面上的点满足 **到球心的距离等于半径**
把射线方程 P = o + t·d 代入球面方程
展开后是一个关于 t 的 **一元二次方程**
令 op 等于球心减去射线起点
判别式 det 等于 b 的平方减去 op 点积自身再加上半径的平方
det 小于零，射线和球不相交
det 等于零，射线与球相切
det 大于零，有两个交点
两个解是 b 加减根号 det


>>> 求交代码 #B12
@enter: fade
@exit: fade
@visual: image

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "intersect 方法实现"，字号 48px，粗体，白色，距顶 50px。
代码块居中，背景 #161b22，圆角 12px，宽度 85%，字号 24px，等宽字体，语法高亮：
  double intersect(const Ray &r) const {
    Vec op = p - r.o;                    // 球心到射线起点
    double t, eps = 1e-4;
    double b = op.dot(r.d);              // 投影长度
    double det = b*b - op.dot(op) + rad*rad; // 判别式
    if (det < 0) return 0;              // 不相交
    det = sqrt(det);
    // 取较小的正值（更近的交点）
    return (t=b-det) > eps ? t :
           ((t=b+det) > eps ? t : 0);
  }
代码块下方注释说明，字号 24px，颜色 #8b949e：
  "eps = 1e-4 的作用：防止射线从表面出发时打到自己（自相交）"
  "返回 0 表示未命中（调用方检查 d > 0）"

--- narration ---
代码和公式完全对应
先算 op 等于球心减去射线起点
b 是 op 在射线方向上的投影
判别式 det 等于 b 的平方减 op 点积自身再加半径的平方
小于零直接返回 0，表示未命中
否则算两个解：b 减 det 和 b 加 det
先试较小的 t，如果大于 **eps** 就取它
否则试较大的 t
eps 是一个微小的正数，10 的负 4 次方
它的作用是 **防止自相交**
从某个表面弹射出去的射线
由于浮点误差可能 t 极小就打回同一个点
eps 排除了这些虚假的交点

代码块居中，背景 #161b22，圆角 12px，宽度 85%，字号 26px，等宽字体：
  inline bool intersect(const Ray &r, double &t, int &id) {
    double n = sizeof(spheres)/sizeof(Sphere);
    double d, inf = t = 1e20;
    for (int i = int(n); i--;)
      if ((d = spheres[i].intersect(r)) && d < t) {
        t = d;
        id = i;
      }
    return t < inf;
  }
代码块下方说明，字号 24px：
  左侧 "输入: 射线 r"
  右侧 "输出: 最近距离 t，命中球体索引 id"
  底部 "遍历所有 9 个球，保留最小 t 对应的球"，颜色 #8b949e。

--- narration ---
全局 intersect 函数做的事情很直接
遍历场景中所有 9 个球
对每个球调用它自己的 intersect 方法
如果返回值 d 大于零（命中）且小于当前最小 t
就更新 t 和 id
循环结束后
t 是射线打到的 **最近的表面** 的距离
id 是那个表面对应的球的索引
返回值表示射线是否命中了任何物体
只有 9 个球，暴力遍历就够了
不需要任何加速结构
