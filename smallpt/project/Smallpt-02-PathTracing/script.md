  Vec nl = n.dot(r.d) < 0 ? n : n*-1;  // 朝向射线的法线
  Vec f = obj.c;                   // 表面颜色
代码块下方，示意图占 60% 宽度，居中：
  一个圆（球截面），圆心标 "p"，表面一点标 "x"。

  另一条射线从内部射出（折射情况），标 "r.d"。

--- narration ---
x 是射线和球体的交点
n 是从球心指向交点的 **几何法线**
对球来说，这个方向永远朝外
但射线可能从球外面打进来，也可能从球里面打出去
从外面打进来时，n 和 r.d 方向相反，dot 为负
从里面打出去时，n 和 r.d 方向相同，dot 为正
**nl** 是"朝向射线来源方向"的法线
如果 dot 为负，说明法线已经朝着射线来的方向，保持不变
如果 dot 为正，说明射线从内部射出，法线要 **翻转**
这个修正对折射计算至关重要

右栏 "怎么做到无偏？"，字号 28px，accent 色标题：
  "• p = 颜色最大分量（存活概率）"，字号 24px
  "• 随机数 < p → 存活，f 除以 p 补偿"，字号 24px
  "• 随机数 ≥ p → 终止，返回自发光"，字号 24px

--- narration ---
路径追踪的光线理论上可以无限弹射
必须有个终止条件
最简单的做法是限制最大深度
但这会引入 **系统偏差**
Russian Roulette 是无偏的替代方案
它的思路是：每次弹射前掷骰子
以概率 p 存活，以概率 1-p 终止
存活时，把贡献除以 p 来 **补偿**
数学上可以证明，这样期望值不变
p 取颜色的最大分量是个好选择
颜色越暗的表面，光线越容易被终止
这符合物理直觉：暗色吸收更多光
代码里 depth 大于 5 才启动 Russian Roulette
前 5 次弹射无条件执行，保证基本质量

  "渲染方程简化为:"
  "L_o = L_e + (albedo/π) · ∫ L_i(ω_i) · cosθ_i dω_i"
  "MC 估计（重要性采样 p(ω) = cosθ/π）:"
  "L_o ≈ L_e + albedo · L_i(ω_sample)"
最后一行用 accent 色高亮。
底部注释 "采样按 cosθ 分布后，π 和 cosθ 全部约掉了！"，字号 22px，颜色 #8b949e。

--- narration ---
漫反射也叫 **Lambertian 反射**
入射光被均匀散射到半球所有方向
BRDF 是常数：反照率除以 π
代入渲染方程后，积分变成
反照率除以 π 乘以半球上 L_i 乘 cos theta 的积分
用蒙特卡洛估计时，如果 **按 cosθ/π 分布采样**
采样的概率密度恰好和被积函数的 cosθ/π 抵消
这叫 **重要性采样**
抵消之后，估计器变得极度简洁
**L_o 约等于 L_e 加上 albedo 乘以采样方向的 radiance**
所有的 π 和 cosθ 都消失了
这就是代码里没有显式除以 π 的原因


>>> 余弦加权半球采样 #B06
@enter: fade-up
@exit: fade
@visual: animation

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "余弦加权半球采样"，字号 48px，粗体，白色，距顶 50px。
中部分为左右两栏，间距 40px：
左栏标题 "数学推导"，字号 28px，accent 色：
  公式列表，字号 24px，行距 36px：
  "目标分布: p(θ,φ) = cosθ · sinθ / π"
  "累积分布函数 CDF:"
  "  φ 均匀: φ = 2π · ξ₁"
  "  θ 按 cosθsinθ: cos²θ = 1 - ξ₂"
  "即: cosθ = √(1-ξ₂), sinθ = √ξ₂"
  "笛卡尔坐标:"
  "  x = cos(2πξ₁) · √ξ₂"  accent 色
  "  y = sin(2πξ₁) · √ξ₂"  accent 色
  "  z = √(1-ξ₂)"          accent 色
右栏标题 "对应代码"，字号 28px，accent 色：
  代码块，背景 #161b22，字号 22px：
    double r1 = 2*M_PI*erand48(Xi); // φ = 2πξ₁
    double r2 = erand48(Xi);         // ξ₂
    double r2s = sqrt(r2);           // √ξ₂ = sinθ
    // 方向 d 的三个分量:
    // u 方向: cos(r1) * r2s  → x
    // v 方向: sin(r1) * r2s  → y
    // w 方向: sqrt(1-r2)     → z (cosθ)

--- narration ---
怎样生成一个按 cosθ 分布的随机方向？
用 **逆变换采样法**
取两个 0 到 1 的均匀随机数 ξ₁ 和 ξ₂
方位角 φ 直接等于 2π 乘以 ξ₁，均匀分布在圆周上
极角 θ 的分布是 cosθ·sinθ
求逆后得到 **cosθ 等于根号 (1 - ξ₂)**
转成笛卡尔坐标
x 等于 cos φ 乘以 sin θ
y 等于 sin φ 乘以 sin θ
z 等于 cos θ
代码里 r1 就是 φ，r2 是 ξ₂
r2s 是根号 ξ₂，就是 sinθ
sqrt(1-r2) 就是 cosθ


>>> 构建局部坐标系 #B07
@enter: fade-up
@exit: fade
@visual: animation

--- visual ---
深色背景 (#0d1117)，内容占画布 90%：
顶部标题 "构建法线方向的正交基"，字号 48px，粗体，白色，距顶 50px。
代码块，背景 #161b22，圆角 12px，宽度 80%，字号 24px：
  Vec w = nl;  // 法线方向 = 局部 z 轴
  Vec u = ((fabs(w.x)>.1 ? Vec(0,1) : Vec(1)) % w).norm();
  Vec v = w % u;
  Vec d = (u*cos(r1)*r2s + v*sin(r1)*r2s + w*sqrt(1-r2)).norm();
代码块下方，示意图，居中，宽度 60%：
  表面一点，三个正交箭头：
  w 朝上（法线方向），标注 "w = nl（法线）"
  u 向右，标注 "u（切线方向 1）"
  v 向前，标注 "v（切线方向 2）"
  一条弯曲箭头从表面射出，偏向法线方向，标注 "采样方向 d"。
底部注释，字号 22px，颜色 #8b949e：
  "选辅助向量时避开与 w 平行的方向: w.x 大选 (0,1,0)，否则选 (1,0,0)"

--- narration ---
采样公式给出的方向是在 **局部坐标系** 里的
z 轴朝法线方向，x y 在切面上
我们需要把这个局部方向转换到世界坐标
方法是构建一组以法线为 z 轴的 **正交基** u v w
w 就是法线 nl
u 用一个辅助向量和 w 叉积得到
辅助向量的选择要避免和 w 接近平行
如果 w.x 的绝对值大于 0.1，就用 (0,1,0)
否则用 (1,0,0)
v 再由 w 和 u 叉积得到
最终方向 d 等于 u 乘以 x 分量加 v 乘以 y 分量加 w 乘以 z 分量
这就完成了从局部到世界的 **坐标变换**

  "令 η = n₁/n₂ (折射率比)"
  "分解入射方向为切线+法线分量:"
  "  d = d_⊥ + d_∥"
  "  d_⊥ = d + (d·n)n  (切线分量)"
  "  d_∥ = -(d·n)n     (法线分量)"
  "折射方向:"
  "  t_⊥ = η · d_⊥"
  "  t_∥ = -√(1 - η²sin²θ₁) · n"  accent 色
  "  cos²θ₂ = 1 - η²(1 - cos²θ₁)"
右栏标题 "对应代码"，字号 28px，accent 色：
  代码块，背景 #161b22，字号 22px：
    double nc = 1;       // 空气折射率
    double nt = 1.5;     // 玻璃折射率
    double nnt = into ? nc/nt : nt/nc; // η
    double ddn = r.d.dot(nl);          // cosθ₁
    double cos2t = 1 - nnt*nnt*(1-ddn*ddn);
    // cos2t < 0 → 全内反射
    Vec tdir = (r.d*nnt - n*((into?1:-1)
      *(ddn*nnt+sqrt(cos2t)))).norm();

--- narration ---
Snell 定律说 n1 sin θ1 等于 n2 sin θ2
我们要算出折射方向的具体向量
关键是把入射方向分解为 **切线分量和法线分量**
切线分量乘以折射率比 η
法线分量用新的 cosθ₂ 重新计算
代码里 nnt 就是 η
into 判断光线是从外面进入玻璃还是从里面出来
如果从外面进入，η 等于 1 除以 1.5
如果从里面出来，η 等于 1.5 除以 1
**cos2t** 是 cos²θ₂
如果它小于零，说明 sin²θ₂ 大于 1
这就是全内反射，没有折射方向

  double a = nt-nc, b = nt+nc;
  double R0 = a*a / (b*b);                   // 垂直入射反射率

每次光线碰到玻璃表面
理论上需要同时追踪 **反射和折射** 两条路径
但如果每次都分裂成两条
路径数量会指数增长
smallpt 的策略是 **分深度处理**
前两次弹射（depth 小于等于 2）
同时追踪两条路径，按 Fresnel 系数加权
这保证了最初几次弹射的精度

**随机选择** 只追踪一条
选反射的概率 P 和 Fresnel 反射率相关
选中哪条，就除以对应的概率来补偿
这又是 **Russian Roulette 的思想**
随机选但补偿权重，保持无偏
